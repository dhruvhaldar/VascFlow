## 2023-10-27 - FastAPI Security Headers & Swagger Documentation
**Vulnerability:** Missing default security headers (`X-Content-Type-Options`, `X-Frame-Options`, `Strict-Transport-Security`, `Content-Security-Policy`). This leaves standard endpoints susceptible to clickjacking, MIME-sniffing, and XSS without layered defense.
**Learning:** Adding a global strict `Content-Security-Policy: default-src 'self'` middleware directly to the FastAPI `app` inherently breaks `/docs` (Swagger UI) and `/redoc` because they fetch static assets (CSS, JS) from external CDNs (like `unpkg.com` or `jsdelivr.net`) and rely on inline scripts.
**Prevention:** When injecting a strict `Content-Security-Policy` via middleware in FastAPI, always condition the CSP application to explicitly exclude the API documentation routes (e.g., `request.url.path.startswith(("/docs", "/redoc", "/openapi.json"))`).
## 2023-10-27 - Application Layer File Size Limits
**Vulnerability:** Missing input length limits (DoS risk) on the `/process_mesh` upload endpoint. Although ASGI servers might have underlying limits, the application didn't enforce a business-logic limit, leaving it susceptible to disk or memory exhaustion from maliciously large file uploads.
**Learning:** FastAPI's `UploadFile` exposes a `.size` attribute, but depending on the ASGI server and how headers are parsed (e.g. `Content-Length`), this attribute can sometimes be `None`. When validating file sizes natively in FastAPI using this attribute, you must explicitly handle the `None` case (`if file.size is not None and file.size > LIMIT:`) to prevent runtime exceptions while still providing a layer of defense.
**Prevention:** Always implement application-layer file size limits on upload endpoints. Furthermore, when writing tests using the FastAPI `TestClient`, do not send actual multi-megabyte payloads; instead, use `unittest.mock.patch` to mock the `.size` property on the `UploadFile` initialization or monkeypatch the specific object inside the endpoint to ensure the test is fast and doesn't exhaust local disk/memory.
